## üìã Customization Guide

### 1. Personal Information
Edit [`index.html`](index.html) to update:
- Your name and title
- Professional summary
- Statistics (years of experience, projects, CVEs, certifications)
- Contact information (email, LinkedIn, GitHub, Twitter)

### 2. Projects
Replace the example projects with your actual work:
- Update project titles and descriptions
- Add real GitHub links and live demos
- Modify technology tags to match your stack
- Add project screenshots (optional)

### 3. Skills
Customize the skills section:
- Adjust skill proficiency percentages
- Add/remove programming languages
- Update security tools based on your expertise
- List your actual certifications

### 4. Research/Blog
Add your security research and writeups:
- Link to your blog posts
- Add vulnerability disclosures
- Include conference talks
- Share security advisories

### 5. Color Scheme
Modify [`styles.css`](styles.css) CSS variables:
```css
:root {
    --primary-color: #00ff41;  /* Change to your preferred color */
    --accent-color: #ff0055;
    --bg-dark: #0a0e27;
}
```

## üì¶ Deployment Options

### Option 1: GitHub Pages (Free)
1. Create a GitHub repository
2. Push your code
3. Go to Settings ‚Üí Pages
4. Select main branch as source
5. Your site will be live at `https://yourusername.github.io/repo-name`

### Option 2: Netlify (Free)
1. Sign up at [netlify.com](https://netlify.com)
2. Drag and drop your project folder
3. Get instant deployment with custom domain support

### Option 3: Vercel (Free)
1. Sign up at [vercel.com](https://vercel.com)
2. Import your GitHub repository
3. Automatic deployments on every push

### Option 4: AWS S3 + CloudFront
1. Create S3 bucket
2. Enable static website hosting
3. Upload files
4. Configure CloudFront for HTTPS and CDN
